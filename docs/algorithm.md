# Topic Scoring Algorithm 개선

## 1. 문제 정의

기존 방식은 전체 문장을 하나로 embedding한 뒤 cosine similarity를 한 번만 계산하여
주제 적합성을 판단하는 구조였다.

이 방식에는 다음과 같은 문제가 있었다:

- 중간에 주제와 무관한 문장이 포함되어도 전체 embedding에 묻혀 감지되지 않음
- 문장별 분석은 존재했지만 실제 점수에는 반영되지 않음
- cosine similarity는 상대값인데, 이를 절대 threshold(0.3)로 판단하여 오차 발생
- 한국어 특성상 무관한 문장도 cosine 값이 0.3 근처로 나와 false positive 발생

---

## 2. 개선 방향

문제의 핵심은 "문장 단위 정보가 전체 평균에 묻힌다"는 점이었다.

따라서 전체 문장이 아닌, **문장 단위 기반 분석 + 집계 방식**으로 구조를 변경하였다.

---

## 3. 개선 방법

### 3.1 Sentence-level 분석

문서를 문장 단위로 분리한 뒤,
각 문장과 주제(anchor) 간의 cosine similarity를 개별적으로 계산하도록 변경하였다.

→ 중간에 이탈하는 문장을 직접 감지할 수 있도록 개선

---

### 3.2 Aggregation (집계 방식 개선)

단순 평균 대신 다음 요소를 함께 고려하도록 변경하였다:

- 평균값 (topic_avg): 전체적인 주제 적합성
- 최소값 (topic_min): 가장 이탈한 문장
- coherence: 문장 간 흐름의 자연스러움

→ 단순히 "대체로 맞다"가 아니라  
→ "이상한 문장이 있는지"까지 반영 가능

---

### 3.3 Coherence (문맥 흐름 반영)

인접 문장 간 cosine similarity를 계산하여
문맥이 자연스럽게 이어지는지를 점수에 반영하였다.

→ 주제는 맞지만 흐름이 깨지는 경우 감지 가능

---

### 3.4 Penalty (이탈 문장 감점)

일정 threshold 이하의 문장을 off-topic으로 간주하고,
그 비율만큼 점수를 감점하도록 추가하였다.

→ 일부 문장이 크게 벗어나는 경우 강하게 패널티 부여

---

### 3.5 Dynamic Normalization

기존:
- 고정된 SIM_MIN / SIM_MAX 사용

문제:
- cosine 값은 데이터마다 분포가 다름

개선:
- 문장 similarity 분포에서 percentile (p5, p95)를 사용하여 동적 정규화

→ 도메인 및 주제에 따라 점수 왜곡 방지

---

### 3.6 On-topic 판단 로직 개선

기존:
- topic_avg >= 0.3 (단일 조건)

문제:
- false positive 발생

개선:
다음 3가지 조건을 모두 만족해야 on-topic:

1. 평균 유사도 >= threshold
2. 최소 유사도 >= 기준값
3. off-topic 문장 비율 < 기준값

→ 단순 평균이 아닌 "전체 구조" 기반 판단

---

## 4. 결과

기존 방식에서는:
- 일부 문장이 주제에서 벗어나도 감지되지 않았음

개선 후:
- 가장 이탈한 문장을 기준으로 문제 문장 탐지 가능
- 문장 흐름(coherence)을 고려한 더 자연스러운 평가 가능
- false positive 감소 (무관한 문장이 통과하는 문제 완화)

---

## 5. 핵심 요약

기존:
- 전체 문장 1회 비교 (global similarity)

개선:
- 문장 단위 similarity 계산
- 평균 + 최소값 + 흐름 + 이탈 비율을 함께 고려

→ 단순 유사도 측정이 아닌  
→ **문장 구조 기반 주제 적합성 평가로 확장**
